# 吉他学习平台需求文档 v1.1

**整理：小霞 · 2026-04-25**

---

## 一、项目概述

**项目名称：** 吉他学习平台 v2  
**定位：** 哥的专属吉他学习与练习工具  
**核心目标：** 围绕两个最高频场景，提供稳定、清晰、顺手的使用体验：

1. **看视频学习**
   - 包括系统课程视频
   - 包括收藏的小红书 / B 站 / YouTube 视频
   - 重点是按课程或来源分类清晰、能快速定位到要学的内容

2. **跟谱和伴奏练歌曲**
   - 看谱顺手
   - 伴奏播放稳定
   - 支持反复练习某一段

**技术栈：**
- 前端：Vue3 + Vite
- 后端：FastAPI
- 本地音频转录：Whisper
- 视频下载：yt-dlp
- PDF 渲染：PDF.js
- GP 渲染：alphaTab

## 二、产品结构

虽然内容来源上仍然分为“课程 / 歌曲 / 收藏视频”，但产品使用体验以两个核心工作流为主：

### 2.1 视频学习

统一承载：
- 系统课程视频
- 收藏视频

核心诉求：
- 分类清楚
- 来源明确
- 定位快速
- 播放稳定

### 2.2 歌曲练习

围绕谱面和伴奏展开：
- 快速找到歌
- 快速切版本
- 打开谱面
- 播放伴奏
- A-B 循环反复练习

### 2.3 AI 助教（增强层）

AI 不是第一优先级主流程，而是附加在学习和练习中的增强能力：
- 课程问答
- 练习任务生成
- 陪练建议

## 三、核心场景一：视频学习

### 3.1 目标

用户进入平台后，可以快速按“课程体系”或“视频来源”定位到想学的内容，并顺畅观看。

### 3.2 内容范围

视频学习包含两类内容：

1. **系统课程视频**
   - 如“吉他新思维”
   - 有明确课程结构
   - 每节课可关联 transcript、曲谱、伴奏、练习任务

2. **收藏视频**
   - 来源于小红书、哔哩哔哩、YouTube
   - 初期以“能下载、能播放、能分类”为主
   - 后续再补结构化字段清洗

### 3.3 分类方式

#### 系统课程视频

至少支持以下层级：

```text
课程体系 / 系列 / 单节课
```

示例：

```text
吉他新思维
  ├── 入门篇
  │   ├── 第1课
  │   ├── 第2课
  │   └── 第3课
  └── 和弦篇
      ├── 第1课
      └── 第2课
```

#### 收藏视频

至少支持以下分类维度：
- 平台来源：小红书 / B站 / YouTube
- 作者
- 主题（如扫弦、和弦、Solo、乐理）
- 关联歌曲（如适用）

初期允许不是所有字段都自动识别，但结构上要预留。

### 3.4 视频学习页的核心要求

#### 列表侧重点

最重要的是：
- **按课程或来源分类**
- **结构清晰**
- **快速定位**

因此列表页至少要满足：
- 左侧或顶部有清晰分类结构
- 当前所在位置明确
- 可以快速切到目标视频

建议支持：
- 课程树 / 分组列表
- 平台来源筛选
- 搜索标题
- 最近观看（可后置）

### 3.5 播放能力

第一阶段只要求：
- 播放稳定
- 支持暂停 / 拖动
- 支持正常切换视频

后续可补：
- 记住播放进度
- 倍速播放
- 快捷键

### 3.6 系统课程附属资料

每节课程可以包含：
- `video.mp4`：课程视频
- `transcript.md`：Whisper 自动转录文本
- `tab.pdf`：附带曲谱（如有）
- `backing.mp3`：附带伴奏（如有）
- `practice.md`：练习任务（可选）

### 3.7 小霞在课程中的角色

#### 听课中

用户可以随时提问，小霞基于当前课程 transcript 回答。

当前原则：
- 优先基于当前课程内容回答
- 如果课程里没有直接答案，可以结合通用吉他知识补充
- 若补充了课程外知识，需要明确说明

#### 听课后

小霞可以生成练习任务，辅助课后练习。

## 四、核心场景二：歌曲练习

### 4.1 目标

用户可以围绕一首歌快速完成以下动作：

```text
找到歌曲 → 选择版本 → 打开谱面 → 播放伴奏 → 循环练习某段
```

### 4.2 入库流程

```text
哥手工整理文件 → 放到指定位置 → 平台扫描入库
```

### 4.3 支持格式

| 格式 | 说明 |
|------|------|
| PDF | 曲谱文件，支持全屏浏览 |
| GP5/GPX | GuitarPro 谱，用 alphaTab 渲染 |
| MP3 | 伴奏音频 |

### 4.4 多版本支持

同一首歌支持多个版本，例如：
- 录音室版
- Live 版
- 教学版

版本需要在歌曲详情页明确展示，并且可以快速切换。

### 4.5 练歌能力优先级

以下优先级已确认，不需要调整：

1. **看谱顺手**
2. **伴奏播放稳定**
3. **A-B 循环**
4. **变速**
5. **打点标记**
6. **多版本切换**

这意味着实现时应优先保证：
- 谱面打开稳定
- 谱面显示清晰
- 音频不乱跳、不失效
- A-B 循环准确可用

### 4.6 播放功能

| 功能 | 说明 |
|------|------|
| 播放/暂停 | 标准音频控制 |
| A-B 循环 | 设置起点、终点，循环播放指定段落 |
| 速度调节 | 0.1x ~ 1.0x，按 0.1 递进 |
| 打点标记 | 在指定时间点添加标记，如“尾奏入口” |

### 4.7 曲谱浏览

需要支持：
- PDF 全屏浏览
- GP 全屏浏览
- 全屏下底部悬浮伴奏控制条
- 控制条层级高于谱面，不被遮挡

### 4.8 MVP 验收重点

歌曲练习模块的 MVP 完成标准不是“功能名义上存在”，而是：

1. 用户能快速进入目标歌曲
2. 用户能顺利打开谱面
3. 用户能稳定播放伴奏
4. 用户能使用 A-B 循环练同一段

### 4.9 数据结构

建议统一为如下结构：

```json
{
  "id": "song_001",
  "title": "灰色轨迹",
  "artist": "Beyond",
  "path": "Beyond/灰色轨迹",
  "versions": [
    {
      "name": "录音室版",
      "files": {
        "gp": "录音室版/track.gp5",
        "pdf": "录音室版/track.pdf",
        "audio": "录音室版/backing.mp3"
      }
    },
    {
      "name": "91 Live版",
      "files": {
        "gp": "91 Live版/track_live.gp5",
        "audio": "91 Live版/backing_live.mp3"
      }
    }
  ],
  "markers": [
    {
      "version": "录音室版",
      "time": 12.5,
      "label": "尾奏入口"
    }
  ]
}
```

## 五、模块三：收藏视频

### 5.1 支持平台

- 小红书
- 哔哩哔哩
- YouTube

### 5.2 核心流程

```text
哥发链接 → 后端下载 → 入库 → 分类展示 → 播放学习
```

### 5.3 第一阶段目标

先做到：
- 能提交视频链接
- 后端完成下载
- 成功入库
- 在前端可播放
- 保留原始标题和来源平台

### 5.4 第二阶段优化

逐个平台补结构化字段：
- 作者
- 歌曲名
- 主题
- 标签
- 无效字符清理

### 5.5 与视频学习页的关系

收藏视频不单独追求复杂系统，而是应最终融入“视频学习”总体验中：
- 来源不同
- 使用体验尽量统一

## 六、AI 助教

### 6.1 课程问答

输入：
- 当前课程
- 当前 transcript
- 用户问题

输出：
- 中文回答
- 优先基于课程内容
- 可补充通用吉他知识，但要说明

### 6.2 练习任务

输入：
- 课程主题
- 难度等级

输出：
- 5 到 8 条循序渐进练习任务
- 练习目标明确
- 附带 2 到 3 条实用建议

### 6.3 陪练能力（后续）

后续可扩展：
- 记录今日练习内容
- 跟进昨天练习进度
- 根据难点动态调整练习建议

## 八、实现原则

### 8.1 优先做顺手，而不是做全

优先把最高频动作打磨好：
- 快速找到视频
- 快速播放视频
- 快速打开谱
- 稳定播放伴奏

### 8.2 优先做统一的数据与路径模型

必须先统一：
- 媒体文件路径规则
- 扫描规则
- 索引字段结构

否则前后端会持续返工。

### 8.3 AI 功能是增强，不抢主流程优先级

AI 要服务“学视频”和“练歌曲”，不应先把大量复杂度堆在主链路之前。

## 九、技术难点

| 难点 | 解决方向 |
|------|---------|
| GP 编码乱码 | utf-8 → latin-1 兜底 |
| PDF 全屏 + 控制条叠加 | 固定底部控制条 + 高 z-index |
| A-B 循环 + 速度协同 | 速度切换后同步 currentTime，并确保循环点逻辑稳定 |
| 多版本目录分组 | 扫描器按子目录识别版本 |
| 课程与收藏视频统一体验 | 统一视频数据结构，来源字段单独区分 |
| 服务稳定性 | 启动脚本、配置管理、日志输出 |
