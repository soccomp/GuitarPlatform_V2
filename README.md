# Guitar Learning Platform v1.1.0

一个面向个人学习场景的本地吉他学习平台。

这个项目最特别的地方，不只是功能本身，而是它的诞生方式：一位编程基础为 0 的吉他练习者，基于自己的真实学习和练琴需求，在 Codex + GPT-4.5 的协作下，用一天半时间完成了第一版正式可用产品。

它不是一个公开内容平台，而是一个围绕“看视频学习”和“跟谱练歌”构建的本地学习工作台。

## 项目目标

这个平台解决的是非常具体的个人学习问题：

- 系统教材、学习视频、歌曲谱和伴奏分散在不同地方，不方便统一管理
- 练一首歌时，需要在播放器、文件夹、PDF、GP 谱之间来回切换
- 收藏了很多 B 站、小红书、YouTube 学习视频，但很难沉淀成自己的长期资料库
- 希望把“学习”和“练习”整合成一个真正适合自己每天使用的平台

因此，整个产品被整理成三个最核心的模块：

- 系统教材
- 学习视频
- 歌曲练习

## 核心功能

### 1. 系统教材

- 支持多套教材并行管理
- 树状章节结构浏览
- 固定教材切换入口
- 课程视频播放与进度记录
- 随课资料整合展示
- 小霞问答与练习任务生成

### 2. 学习视频

- 扫描本地收藏视频目录
- 使用卡片流展示视频内容
- 自动生成真实视频封面
- 弹窗播放
- 竖屏视频自适应
- 搜索、最近打开、重新扫描
- 平台内直接删除视频并同步清理缩略图

### 3. 歌曲练习

- 歌曲列表与多版本管理
- PDF / GP 谱打开与预览
- MP3 伴奏播放
- 进度条拖动
- A/B 打点循环
- `0.60x - 1.00x` 速度调节
- 标记功能
- 谱子弹窗底部紧凑播放器

## 平台架构

- Vite

后端：

- FastAPI

- `library/courses/`

本地服务：

- 前端：`http://127.0.0.1:3000/`
- 后端：`http://127.0.0.1:8000/`

## 一个很有代表性的创新点

用户不需要手动维护视频索引文件。

当用户把 B 站、小红书、YouTube 的视频链接发给 `openclaw` 的 agent 后，agent 可以：

1. 下载视频
2. 把视频存入 `library/` 目录
3. 平台自动扫描本地素材
4. 新内容自动进入“学习视频”模块

这让“导入学习资料”从一个手工后台操作，变成了一种更自然的 AI 协作工作流。

## 本地运行

日常访问：

```text
http://127.0.0.1:3000/
```

平台已经支持本地后台常驻运行。桌面配套有：

- `吉他学习平台启动器.app`
- `吉他学习平台服务控制器.app`

如需脚本方式启动：

```bash
scripts/start_guitar_platform.command
```
